# x402 / USDC

The [x402](https://github.com/0xKoda/x402-wallet/blob/main/wallet.md) rail settles a **USDC payment on the base-sepolia testnet** and returns the
`SettlementResponse` + `PaymentRequirements` the gateway extracts the metered cost from —
metering into the **same cross-rail cap as Stripe**. This is the second rail and the moat.

!!! warning "x402 needs a funded wallet + facilitator — not just a key"
    Unlike Stripe, x402's live on-chain settle needs **a funded base-sepolia USDC testnet
    wallet *and* an x402 facilitator URL**. A key alone does **not** fund the transfer. This
    is real setup; budget for it.

## Prerequisites

- A **base-sepolia** wallet **funded with testnet USDC** (private key, `0x…`). Fund it from
  a base-sepolia ETH faucet (for gas) and a testnet USDC faucet.
- An **x402 facilitator URL** (the service that brokers the settle).

!!! danger "Testnet only"
    A **mainnet** network is **refused** — this rail is test-money only.

## Setup

```bash
# 1. self-check (no wallet): confirm the adapter extracts the cost the gateway probe expects
node examples/payments/adapters/x402-adapter/test.mjs

# 2. run live on testnet, with the wallet + facilitator custodied by the gateway
export X402_WALLET_PRIVATE_KEY=0x...        # a FUNDED base-sepolia USDC wallet
export X402_FACILITATOR_URL=https://...     # the x402 facilitator
auths-mcp wrap \
  --scope paid.call --budget '$5' --ttl 30m \
  --custody-credential X402_WALLET_PRIVATE_KEY \
  --custody-credential X402_FACILITATOR_URL \
  -- node examples/payments/adapters/x402-adapter/server.mjs
```

## How it works

```
agent ──tools/call (x402)──▶ gateway ──tools/call──▶ x402-adapter ──▶ facilitator (base-sepolia)
                                │  reserve ceiling BEFORE the rail, ACROSS rails  ◀── SettlementResponse
                                │  extract requirements.maxAmountRequired (atomic USDC → cents)
                                │  settle into the SAME cap the Stripe rail meters into
                                ▼
                             receipt: rail=x402, tx=0x…, cross-rail total
```

### atomic-USDC → cents

USDC has **6 decimals**, so x402 amounts are **atomic**: `1500000` atomic = `1.50` USDC =
**150 cents**. The conversion is `atomic / 10000`, **exact only** — a sub-cent residue is
refused, not truncated, so the metered cost equals the settled amount exactly. The receipt
names the on-chain settlement **tx** (`0x…`).

### The cross-rail moat

A `$1.50` Stripe charge **+** a `$1.50` x402 settle = `$3.00` of one `$5` cap. A later
**`$0.60`** x402 settle is tiny in a per-rail x402 silo (x402-alone is only `$2.10`) — but
summed cross-rail onto `$4.50` it would reserve to `$5.10 > $5`, so the gateway refuses it
`usage-cap-exceeded` **before** the adapter settles. Two siloed per-rail budgets would each
wave it through; the one cross-rail counter refuses it.

## Never faked

With no wallet + facilitator the adapter runs **hermetic** (the recorded testnet settlement
shape, not a fabricated on-chain settle). If the live path is reached without a reachable
facilitator it **throws** rather than inventing a transaction.

## What's proven without a wallet vs. what the wallet finishes

- **Proven hermetically (no wallet, gated):** the gateway extracts the amount from the
  recorded x402 response (atomic → cents), meters it into the cross-rail cap, refuses the
  cross-rail cap-crosser before the rail, and names `0x…` + `rail=x402` in the receipt.
- **Finished by the funded wallet + facilitator:** the **live** USDC settle on
  base-sepolia — the real on-chain leg.

## Troubleshooting

| Symptom | Cause |
| --- | --- |
| adapter runs but never settles on-chain | no wallet + facilitator → it's in **hermetic** mode |
| live path throws instead of settling | facilitator URL unset/unreachable — by design, it never fabricates a tx |
| mainnet network rejected | by design — testnet only |
| `usage-cap-exceeded` on a tiny x402 call | the cross-rail moat — the shared cap is already spent (expected) |
| sub-cent amount refused | x402 atomic→cents is exact-only; align the amount to whole cents |
